# /audit-site — サイト監査

既存サイトの状態を監査し、結果を日本語でまとめるコマンド。
**修正はしない。** 監査と報告のみ。

## やること

1. 既存の監査スクリプトを探す
   - `scripts/`・`tools/`・`package.json` などから監査系スクリプトを探す。
   - 見つかったら、それを優先して使う（独自に作らない）。

2. npm scripts を確認する
   - `package.json` の `scripts` を読み、`audit` / `check` / `lint` / `validate` 系がないか確認。
   - あれば実行する。無ければ手動チェックに切り替える。

3. 手動チェック項目を確認する
   - 記事数（一覧と実ファイルの整合）
   - sitemap（存在・記事との整合・URL 形式）
   - feed（RSS/JSON feed の存在・整合）
   - PR表記（アフィリエイト記事に開示があるか）
   - canonical（重複・欠落・誤り）
   - 内部リンク（リンク切れ・誤 slug）
   - 画像リンク（存在しない画像・パス誤り）

4. 結果をまとめる
   - **エラー（壊れている / 公開に支障）** と **警告（望ましくない / 要確認）** を分けて日本語で列挙する。
   - 各項目に「対象」「内容」「推奨対応」を添える。

## 禁止事項

- ファイルの修正・生成（レポート以外）
- 破壊的なスクリプト実行

## 最後に

日本語で「監査した項目」「エラー件数 / 警告件数」「次にやるとよいこと（例: /fix-safe で安全修正）」を短く説明する。
