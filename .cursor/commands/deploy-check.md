# /deploy-check — 公開前チェック

push / 公開してよい状態かを確認するコマンド。
**勝手に push しない。** push が必要ならユーザーに確認を求める。

## やること

1. git status 確認
   - `git status` で変更ファイル・未追跡ファイルを確認する。
   - 想定外の変更が混ざっていないか確認する。

2. 差分確認
   - `git diff`（および必要なら `git diff --staged`）で変更内容を確認する。
   - PR表記 / canonical / sitemap / feed / slug / 内部リンク を壊していないか重点的に見る。

3. 監査実行
   - `/audit-site` 相当のチェックを実行する。
   - エラーが残っていないか確認する。

4. push してよい状態か判定する
   - エラーがある / 破壊的変更が疑われる → **push 不可**（理由を明示）。
   - 問題なし → **push 可**（ただし実行はユーザー確認後）。

5. push する場合
   - push は破壊的・外部に効く操作なので、**必ずユーザーに確認を求めてから**実行する。
   - force push は原則行わない。

## 禁止事項

- 無確認の push / force push
- エラーを残したままの公開判定

## 最後に

日本語で「git status の要点」「差分の要点」「監査結果」「公開可否（理由つき）」「次の操作」を短くまとめる。
