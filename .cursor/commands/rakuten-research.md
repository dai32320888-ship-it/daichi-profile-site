# /rakuten-research — 楽天アフィリエイト記事の調査・更新候補出し

楽天アフィリエイト記事の情報収集を行い、更新候補をレポートにまとめるコマンド。
**記事本文は勝手に書き換えない。** まず調査とレポート作成だけを行う。

## やること

1. 既存記事を確認する
   - 楽天アフィリエイト関連の記事ファイル・一覧・データを探して読む。
   - 記事のタイトル・説明文・商品リンク・PR表記・公開日を把握する。

2. 古そうな要素を探す
   - 古い・切れていそうな商品リンク
   - 型落ち・終売の可能性がある商品
   - 古いタイトル / 説明文
   - PR表記が抜けていそうな箇所

3. 楽天商品情報を使う前提で更新候補を出す
   - 「どの記事の」「どこを」「どう更新すべきか」を候補として整理する。
   - 実際の書き換えはしない。あくまで候補出し。

4. レポートを作成する
   - `reports/rakuten-update-candidates.md` にまとめる（フォルダが無ければ作る）。
   - 各候補について次を日本語で書く：
     - 対象記事（ファイル名 / URL / slug）
     - 更新候補（何を変えるか）
     - 理由（なぜ更新が必要か）
     - 注意点（PR表記・canonical・内部リンクを壊さないための注意）

## 禁止事項

- 記事本文・リンクの自動書き換え
- PR表記の削除
- slug / canonical / sitemap / feed の変更

## 最後に

日本語で「何を調べたか」「更新候補が何件あるか」「次にどのコマンドで進めるとよいか（例: /fix-safe）」を短く説明する。
