# /add-article — 記事追加

既存記事と同じ形式で新しい記事を安全に追加するコマンド。
**独自形式を作らない。** 既存の構造を真似る。

## やること

1. 既存記事の構造を確認する
   - 既存記事ファイルを 1〜2本読み、フロントマター・見出し・本文・PR表記の形式を把握する。
   - slug / canonical / 公開日 / タグなどのルールを確認する。

2. 同じ形式で新規記事を追加する
   - 既存記事と同じテンプレート・書式で作成する。
   - タイトル・本文・商品情報などは指示内容に沿って埋める。

3. 関連する登録先を確認・更新する
   - slug が既存と重複しないか
   - canonical が正しいか
   - sitemap に追加が必要か
   - feed に反映されるか
   - 一覧ページに載るか
   - 新着順（公開日順）が正しく並ぶか

4. PR表記を必ず入れる
   - アフィリエイト記事にはPR表記（広告開示）を必ず含める。

5. 内部リンクを確認する
   - 新記事からのリンク・新記事へのリンクが正しいか確認する。
   - 既存の内部リンクを壊していないか確認する。

6. 最後に監査を実行する
   - `/audit-site` 相当のチェックを行い、整合を確認する。

## 禁止事項

- PR表記の省略
- 既存記事の書式を無視した独自形式
- 既存 slug / canonical / sitemap / feed の破壊

## 最後に

日本語で「追加した記事」「更新した登録先（sitemap / feed / 一覧など）」「監査結果」「次にやるとよいこと（例: /deploy-check）」を短く説明する。
