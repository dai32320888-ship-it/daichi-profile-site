# /fix-safe — 安全修正

限定した範囲だけを安全に修正するコマンド。
**いきなり広範囲を直さない。** 現状確認 → 範囲限定 → 修正 → 監査の順で進める。

## やること

1. まず現状確認
   - 何を直すのかを明確にする（対象・目的）。
   - 関連ファイルを読んで、現在の構造・書式を把握する。

2. git status 確認
   - `git status` で今の変更状態を確認する。
   - 想定外の変更が残っていないかチェックする。

3. 変更対象を限定する
   - 直すファイル・箇所を最小限に絞る。
   - 「触らないファイル」を先に決める。

4. 修正する（破壊的変更は禁止）
   - PR表記 / canonical / sitemap / feed / slug / 内部リンクを壊さない。
   - 既存の書式・構成に合わせる。
   - 不明点は推測で変えず、レポートに残す。

5. 修正後に監査する
   - `/audit-site` 相当のチェックを実行し、壊していないか確認する。

## 禁止事項

- 対象外ファイルの変更
- slug / canonical / sitemap / feed / PR表記 の破壊
- 一括置換による意図しない変更

## 最後に

日本語で「変更したファイル一覧」「各ファイルで何を直したか」「監査結果」「次にやるとよいこと（例: /deploy-check）」を短く説明する。
