# 楽天アフィリエイト作業プロンプト

楽天アフィリエイト記事の調査・更新・追加をするときの方針書。
（コマンド版は `/rakuten-research`・`/add-article`）

## 前提

- ユーザーは非エンジニア寄り。説明は日本語で、次の操作を短く示す。
- 記事は既存の書式・構成に合わせる。独自形式を作らない。

## PR表記（最重要）

- アフィリエイト記事には **必ず PR表記（広告開示）** を入れる。
- 景表法・ステマ規制に触れる断定表現は禁止。
- 既存記事の PR表記の書き方に合わせる。

## 調査の進め方（自動書き換えしない）

1. 既存記事を読む（タイトル・説明・商品リンク・PR表記・公開日）。
2. 古そうな要素を洗い出す：
   - 切れていそうな / 古い商品リンク
   - 型落ち・終売の可能性がある商品
   - 古いタイトル・説明文
   - PR表記の抜け
3. 楽天商品情報を使う前提で「更新候補」を出す（書き換えはまだしない）。
4. `reports/rakuten-update-candidates.md` にまとめる：
   - 対象記事（ファイル / URL / slug）
   - 更新候補（何を変えるか）
   - 理由
   - 注意点（壊してはいけない項目）

## 記事追加の進め方

1. 既存記事の構造をコピーする。
2. slug 重複・canonical・sitemap・feed・一覧・新着順を確認。
3. PR表記を入れる。
4. 内部リンクを確認。
5. 監査を実行。

## 壊してはいけないもの

PR表記 / canonical / sitemap / feed / slug / 内部リンク / 画像リンク

## 不明点

推測で破壊的変更をしない。判断できない点は `reports/` に残す。
