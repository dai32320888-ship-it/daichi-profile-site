# サイト運用プロンプト

サイトの監査・修正・運用をするときの方針書。
（コマンド版は `/audit-site`・`/fix-safe`）

## 前提

- 既存サイトを壊さない。動いている構成を最優先で守る。
- 自動修正より安全確認を優先する。

## 監査（/audit-site）

まず修正せず、状態を確認する。

1. 既存の監査スクリプトを探す（`scripts/` / `tools/` / `package.json`）。あれば使う。
2. `package.json` の scripts（audit / check / lint / validate 系）を確認・実行。
3. 手動チェック：
   - 記事数（一覧と実ファイルの整合）
   - sitemap（存在・整合・URL 形式）
   - feed（RSS / JSON feed の整合）
   - PR表記の有無
   - canonical（重複・欠落・誤り）
   - 内部リンク（切れ・誤 slug）
   - 画像リンク（欠落・パス誤り）
4. 結果を **エラー** と **警告** に分けて日本語でまとめる。

## 安全修正（/fix-safe）

1. 現状確認（対象ファイルを読む）。
2. `git status` 確認。
3. 変更対象を最小限に限定。触らないファイルを先に決める。
4. 修正（破壊的変更禁止）。既存書式に合わせる。
5. 修正後に監査。

## 壊してはいけないもの

PR表記 / canonical / sitemap / feed / slug / 内部リンク / 画像リンク

## 不明点

推測で破壊的変更をしない。判断できない点は `reports/` に残す。

## 説明

最後に日本語で「変更したファイル一覧」「何を直したか」「監査結果」「次の操作」を短く示す。
