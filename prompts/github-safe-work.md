# GitHub 安全作業プロンプト

コミット・push・公開まわりを安全に進めるための方針書。
（コマンド版は `/deploy-check`）

## 前提

- push は外部に効く操作。**必ずユーザー確認後**に行う。
- 勝手に push / force push しない。

## 公開前チェックの順序

1. `git status` で変更・未追跡ファイルを確認。想定外の変更が無いか見る。
2. `git diff`（必要なら `git diff --staged`）で差分を確認。
   - PR表記 / canonical / sitemap / feed / slug / 内部リンク を壊していないか重点確認。
3. 監査を実行（`/audit-site` 相当）。エラーが残っていないか。
4. push 可否を判定：
   - エラーあり / 破壊的変更の疑い → **push 不可**（理由を明示）。
   - 問題なし → **push 可**（実行はユーザー確認後）。

## push するとき

- 内容を要約してユーザーに確認を求める。
- OK が出てから push する。
- force push は原則しない。必要な場合も必ず事前確認。

## やってはいけないこと

- 無確認 push / force push
- git config の変更
- エラーを残したままの公開判定
- 秘密情報（.env / 認証情報）のコミット

## 説明

最後に日本語で「git status の要点」「差分の要点」「監査結果」「公開可否（理由つき）」「次の操作」をまとめる。
